> 基于历史打卡记录本（history）页面总结的通用布局规范

---

## 一、整体结构

### 1.1 页面容器
```css
.history-container {
  padding: 20rpx;
  background-color: #f8f8f8;
  min-height: 100vh;
}
```

**规范要点：**
- 背景色：`#f8f8f8`（浅灰）
- 内边距：`20rpx`
- 最小高度：`100vh`

### 1.2 页面模块顺序
```
1. 筛选器（Filters）
2. 统计卡片（Stats）
3. 数据列表/表格（List/Table）
```

|------|--------|------|
| 主渐变 | `#667eea` → `#764ba2` | 表头、按钮、标题栏 |

| 警告/低分 | `#ff4d4f` | 分数<6 |
| 未填写 | `#ccc` | 空数据 |

## 十二、注意事项

1. **一致性**：所有页面使用相同的颜色、字体、间距规范
2. **响应式**：使用百分比和 flex 布局适配不同屏幕
3. **性能**：长列表使用 scroll-view，避免页面卡顿
4. **可访问性**：确保文字大小≥24rpx，颜色对比度足够
5. **弹框层级**：z-index 统一使用 1000 以上
6. **动画时长**：过渡动画控制在 0.3s 内
