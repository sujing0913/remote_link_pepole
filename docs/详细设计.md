# 学习打卡与单词本小程序（remote_link_pepole）— 详细设计

> 本文档面向研发/测试/运维，描述当前项目已实现功能的**详细设计**，包括：模块设计、数据库设计、API/云函数设计、关键流程、异常处理与安全建议。  
> 说明：本项目以微信小程序 + 微信云开发为主，后端 API 形态以**云函数**与**云数据库直连查询**两种方式呈现。

---

## 1. 运行环境与技术栈

### 1.1 客户端
- 微信小程序（基础库 2.12+ 建议）
- 页面：WXML + WXSS + JS
- 能力：`wx.cloud`（云函数、云存储、数据库）、媒体采集、预览、分享

### 1.2 云开发
- 云函数（Node.js）
- 云数据库（NoSQL，集合）
- 云存储（fileID）
- openapi：`cloud.openapi`（OCR、订阅消息等）

### 1.3 第三方服务
- 豆包/火山方舟（在 `doubaoAI` 云函数内调用）
- 腾讯混元（在 `aiEvaluate` 云函数内调用，部分链路）
- 有道词典发音（前端拼接 URL 播放）

## 2. 模块详细设计

### 2.1 小程序页面（miniprogram/pages）

#### 2.1.1 首页 pages/index（学习打卡）
**职责**
- 科目选择（语文/数学/英语）
- 打卡：相机拍摄（图片/视频）→ 上传云存储 → 写入 `check_ins`
- 今日状态：查询今日是否打卡
- 统计：查询指定日期范围累计打卡次数（默认本月）
- AI 评分：仅对“今日最新图片打卡”手动触发，调用 `doubaoAI`，解析 JSON 写回 `check_ins` 并展示弹框

**关键数据字段（data）**
- `subjects`、`selectedSubjectIndex`
- `currentUserOpenId`
- `latestPunch`：首页展示最新打卡记录（含临时 URL）
- `recordId`：本次打卡记录 ID（用于 AI 回写）
- AI 弹框：`showAIResultModal`、`aiScore`、`aiJudgment`、`aiSuggestion`、`aiCheckResults` 等
- 加载动画：`showAILoading`、`currentAnimal`、`animalTimer`

**核心函数（摘录）**
- `onLoad()`：调用 `getUserInfo` 获取 openId；初始化日期；检查今日状态；查询本月累计
- `onPunch()` → `uploadFile()` → `saveToDatabase()`：打卡上传并写库
- `onAIEvaluate()`：查今日最新记录 → 调 `callAIEvaluate()`
- `callAIEvaluate()`：调用云函数 `doubaoAI`，传 fileID + prompt，期望 JSON
- `saveAIResultToDB()`：把 AI 结果字段写回 `check_ins`

**异常/提示**
- 上传失败提示“上传失败”
- 记录查询不到提示“今日还未打卡”
- 非图片媒体提示“仅支持图片评分”
- AI 返回失败提示“AI 分析失败，请重试”

---

#### 2.1.2 历史页 pages/history（记录、统计、查看 AI 结果）
**职责**
- 选择年/月范围（含“全部月份”）
- 科目选择
- 读取 `check_ins`（按时间倒序）并按月分组
- 统计：年度打卡数、总分、奖杯/小红花
  - 总分口径：同一天多次打卡取**最高得分**累加
- 支持查看媒体、建议、检查结果详情弹框

**核心函数**
- `generateYearMonthRange()`：生成可选年份与月份数组
- `fetchRecords()`：构造时间范围 + 科目 + `puncherOpenId` 查询 `check_ins`
- `processRecords()`：格式化时间、按月分组、按天取最高分计算总分

**异常/提示**
- 获取 openId 失败提示
- DB 查询失败提示“加载失败”

---

#### 2.1.3 工具箱 pages/toolbox
**职责**
- 功能入口聚合（拍照识词、单词本等）
- 具体逻辑在对应页面实现

---

#### 2.1.4 拍照识词 pages/scanWord
**职责**
- 选择图片（相机/相册）上传云存储
- 调用 `doubaoAI` 输出结构化单词信息（JSON）
- 展示单词：音标、释义、例句、记忆技巧
- 支持播放发音（有道 dictvoice）

---

#### 2.1.5 单词本 pages/wordbook
**职责**
- 从集合 `wordbook` 拉取单词列表并按月份分组展示
- 新增：手动输入或识别后新增（内部会调用云函数进行分析/结构化）
- 复习：筛选 remembered 状态、编辑 remark、播放发音

## 3. 数据库设计（云数据库）

> 云开发数据库为 NoSQL，以下以“字段+含义+建议索引/约束”描述。  
> 注意：前端中部分集合直接读写（`check_ins`、`wordbook`），因此需要在云开发控制台正确配置权限规则。

### 3.1 users（用户档案）
**用途**
- 存储用户标识与基础信息（openId 冗余存储），为后续扩展用户体系准备
- 当前项目通过 `getUserInfo` 获取 openId；是否写入 users 视云函数实现而定

**字段建议**
| 字段 | 类型 | 说明 |
|---|---|---|
| openid | string | 用户 openId（冗余存储） |
| nickName | string | 昵称（可选） |
| avatarUrl | string | 头像（可选） |
| role | string | 角色（可选扩展） |
| create_time | date | 创建时间 |
| update_time | date | 更新时间 |

> 备注：仓库中新增（但未在前端接入）的 `ensureParentBindCode` 会使用 `users.bind_code` 字段；属于扩展功能。

**索引建议**
- `openid` 唯一索引（强烈建议）

---

### 3.2 check_ins（学习打卡记录）
**用途**
- 保存用户每次打卡的媒体、科目、时间、AI 评分结果等

**字段（结合现有页面使用）**
| 字段 | 类型 | 说明 |
|---|---|---|
| mediaUrl | string | 云存储 fileID（或临时 URL） |
| mediaType | string | `image` / `video` |
| createTime | date | 打卡时间（serverDate） |
| subject | string | 科目：语文/数学/英语 |
| puncherOpenId | string | 打卡用户 openId |
| score | number | -1 未评分；0-10 评分 |
| suggestion | string | 学习建议 |
| aiAnalysis | string | AI 判断/分析 |
| recognizedContent | string | 识别到的内容文本 |
| totalQuestions | number | 题目数 |
| correctQuestions | number | 正确数 |
| checkResults | array | 明细数组：{question,user_answer,correct_answer,is_correct} |
| analyzedAt | date | 分析时间（serverDate） |

**索引建议**
- 复合索引：`puncherOpenId + subject + createTime`（用于首页/历史页高频查询）
- 单字段索引：`createTime`（历史范围查询）

---

### 3.3 wordbook（单词本）
**用途**
- 保存单词条目及其结构化信息与记忆状态

**字段（结合页面实现）**
| 字段 | 类型 | 说明 |
|---|---|---|
| word | string | 单词 |
| phonetic | string | 音标 |
| meaning | string | 释义 |
| sentences | array/string | 例句（实现可为数组或文本） |
| memoryTips | string | 记忆技巧 |
| pronunciationUrl | string | 发音 URL（有道 dictvoice） |
| remembered | boolean | 是否已记住 |
| remark | string | 备注 |
| createTime | date | 创建时间 |
| updateTime | date | 更新时间 |
| ownerOpenId | string | 所属用户 openId（若已实现） |

**索引建议**
- `createTime`
- `word`（用于去重查询）
- 如果做多用户：`ownerOpenId + createTime`

---

### 3.4 （扩展）binds（亲子绑定关系，仓库存在 bindToParent 使用）
> 当前前端未接入，但仓库云函数 `bindToParent` 已使用此集合。

**字段**
| 字段 | 类型 | 说明 |
|---|---|---|
| parent_openid | string | 家长 openId |
| child_openid | string | 孩子 openId |
| create_time | date | 绑定时间 |

**索引建议**
- `parent_openid`
- `child_openid`
- 复合唯一约束（逻辑）：`parent_openid + child_openid` 防止重复绑定

## 4. API / 云函数设计

### 4.1 通用约定
- 入参：`event`（JSON）
- 鉴权：通过 `cloud.getWXContext().OPENID`
- 返回结构建议：
  - `{ success: true, data: ... }`
  - `{ success: false, message: '...' }`

---

### 4.2 getUserInfo（已存在）
**用途**：获取当前用户 openId（及可选 userInfo）。  
**调用方**：首页、历史页等。

**输入**：无或微信上下文。  
**输出**：`{ openId }`（具体字段以实现为准）。

---

### 4.3 doubaoAI（已存在）
**用途**：统一 AI 入口（多模态识别/分析）。  
**调用方**：首页 AI 评分、scanWord、wordAnalyze 等。

**输入（示例）**
- `fileID`：图片云存储 fileID（可选）
- `userPrompt`：提示词（必填）

**输出（示例）**
- `{ success: true, data: <模型返回 JSON 对象> }`

**关键点**
- 需要将 `fileID` 转换为临时 URL 或通过云端方式供模型读取（具体实现见云函数代码）
- 需要对模型输出做 JSON 解析与容错（模型可能返回非严格 JSON）

---

### 4.4 ocrGeneral（已存在）
**用途**：微信 OCR 通用文字识别（备用/扩展）。  
**输入**：图片（fileID/URL）  
**输出**：OCR 文本结构。

---

### 4.5 aiEvaluate（已存在）
**用途**：备用评分链路（可能先 OCR 再调用混元）。  
**说明**：首页当前评分主要走 doubaoAI；该函数用于扩展/备用。

---

### 4.6 wordAnalyze（已存在）
**用途**：对输入文本进行“识词/结构化解释”，内部调用 `doubaoAI`。  
**输入**：`{ text }`  
**输出**：结构化单词信息 + pronunciationUrl。

---

### 4.7 ensureParentBindCode（已存在，扩展）
**用途**：确保家长用户拥有永久 6 位绑定码（如不存在则生成并持久化）。  
**集合**：`users`  
**输入**：无（从上下文取 openid）  
**输出**
- `{ success: true, bind_code, isNew }`

**设计要点**
- 随机生成 6 位数字，检查 `users.bind_code` 唯一（最多重试 20 次）
- 若 users 不存在：创建用户档案并写入 role='parent' 和 bind_code
- 若存在但无 bind_code：补生成并更新
- 若存在且有 bind_code：直接返回（永久不变）

---

### 4.8 bindToParent（已存在，扩展）
**用途**：孩子输入家长绑定码进行绑定。  
**集合**：`users`、`binds`  
**输入**：`{ bind_code }`  
**输出**：绑定成功/失败信息（以实现为准）。

---

### 4.9 getMyBindings（新增，扩展）
**用途**：查询当前用户的绑定关系（作为家长绑定的孩子列表/作为孩子绑定的家长列表）。  
**集合**：`binds`、`users`  
**输入**：无（上下文 openid）  
**输出**
```json
{
  "success": true,
  "data": {
    "asParent": [{"openid":"childOpenId","name":"..." }],
    "asChild": [{"openid":"parentOpenId","name":"..." }]
  }
}
```

---

### 4.10 notifyParentOnPunch（新增，扩展）
**用途**：孩子打卡后向其绑定的家长发送订阅消息提醒。  
**集合**：`binds`、`check_ins`  
**输入**：`{ recordId }`  
**输出**：通知成功数量与逐个发送结果。

**模板**
- 模板 ID（tmplId）：`ZTWzbhWfZxCTXBPLJFKLbmZ89F1b_6tcfUlhEPmFpyA`
- 当前实现按 `keyword1~keyword4` 发送：科目、时间、得分、建议摘要
- 若后台模板关键词命名不同，需要按实际调整 `data` 字段

**注意事项**
- 订阅消息必须由家长端先 `wx.requestSubscribeMessage` 授权（一次性）
- 页面跳转当前配置为 `pages/history/history`

## 5. 关键流程设计（详细）

### 5.1 打卡写库
1. `wx.chooseMedia` 获取 `tempFilePath` 与类型（image/video）
2. `wx.cloud.uploadFile` 上传到云存储，得到 `fileID`
3. `db.collection('check_ins').add` 写入记录，score=-1

### 5.2 AI 评分写回
1. 前端查今日最新记录（subject + puncherOpenId + createTime 范围）
2. 调云函数 `doubaoAI(fileID, prompt)`
3. 解析返回 JSON：score/judgment/suggestion/check_results...
4. 更新 `check_ins` doc：写入 AI 结果字段与 analyzedAt
5. 首页弹框展示；历史页可查看明细

### 5.3 历史统计（按天最高分）
- 遍历记录，以 dayKey=YYYY-MM-DD 聚合，取 score 最大值
- `totalScore = sum(maxScoreByDay)`
- `trophyCount = floor(totalScore / 100)`
- `flowerCount = floor((totalScore % 100) / 10)`

## 6. 权限与安全设计

### 6.1 云数据库权限
当前页面存在前端直连数据库读写（`check_ins`、`wordbook`）。建议：
- **写**：仅允许创建者写自己的数据
- **读**：仅允许创建者读自己的数据
- **更新**：仅允许创建者更新自己的数据（AI 结果更新也在前端执行）

更安全的方式：将写库/查询/更新统一迁移到云函数中，由云函数做权限校验。

### 6.2 第三方密钥管理
- `doubaoAI`、`aiEvaluate` 相关密钥应放在云函数环境变量/云开发配置，不应硬编码到仓库。

### 6.3 订阅消息合规
- 需要用户明确授权（一次性/长期）
- 消息内容应与用户操作/服务相关，不得滥发

## 7. 错误码与异常处理建议

| 场景 | 建议错误信息 | 处理 |
|---|---|---|
| 未获取 openid | 未获取到用户身份 | 终止并提示重试 |
| 上传失败 | 上传失败 | 提示用户重新拍摄/检查网络 |
| DB 写入失败 | 保存失败 | 提示重试，打印 err |
| AI 返回格式错误 | AI 分析失败，请重试 | 增加 JSON 解析容错/回退 |
| 订阅消息发送失败 | 发送订阅消息失败 | 返回失败原因（无权限/未授权/模板字段不匹配） |
