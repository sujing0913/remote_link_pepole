# 用户角色数据库设计文档

## 概述

本文档描述 `remote_link_pepole` 小程序的用户角色系统设计，支持**家长**、**孩子**、**普通用户**三种角色的管理和切换。

## 角色定义

### 角色枚举

| 角色标识 | 角色名称 | 描述 | 图标 |
|---------|---------|------|------|
| `normal` | 普通用户 | 未绑定关系的独立用户 | 👤 |
| `parent` | 家长 | 有绑定码，可绑定多个孩子 | 👨‍👩‍👧 |
| `child` | 孩子 | 通过绑定码绑定到家长 | 👶 |

### 角色权限

| 权限 | 普通用户 | 家长 | 孩子 |
|-----|---------|------|------|
| 可以成为家长 | ✅ | ✅ | ❌ |
| 可以成为孩子 | ✅ | ❌ | ✅ |
| 可以生成绑定码 | ✅ | ✅ | ❌ |
| 默认有绑定码 | ❌ | ✅ | ❌ |

## 数据库设计

### users 集合

存储用户档案信息。

| 字段 | 类型 | 说明 | 索引 |
|-----|------|------|------|
| `_id` | string | 文档 ID | 主键 |
| `_openid` | string | 用户 openid（系统字段） | 唯一索引 |
| `openid` | string | 用户 openid（冗余字段） | 唯一索引 |
| `nickName` | string | 昵称 | - |
| `avatarUrl` | string | 头像 URL | - |
| `role` | string | 角色：normal/parent/child | 索引 |
| `role_update_time` | date | 角色更新时间 | - |
| `bind_code` | string | 6 位绑定码（仅家长） | 唯一索引 |
| `bind_code_create_time` | date | 绑定码创建时间 | - |
| `bind_code_update_time` | date | 绑定码更新时间 | - |
| `create_time` | date | 创建时间 | 索引 |
| `update_time` | date | 更新时间 | - |

### binds 集合

存储用户之间的绑定关系。

| 字段 | 类型 | 说明 | 索引 |
|-----|------|------|------|
| `_id` | string | 文档 ID | 主键 |
| `parent_openid` | string | 家长 openid | 复合索引 |
| `child_openid` | string | 孩子 openid | 复合索引 |
| `status` | number | 状态：0-解绑，1-绑定 | 复合索引 |
| `create_time` | date | 绑定时间 | - |
| `update_time` | date | 更新时间 | - |
| `unbind_time` | date | 解绑时间 | - |

## 云函数设计

### 核心云函数

#### 1. getUserInfo
获取当前用户信息，返回增强的角色信息。

**输入：**
```javascript
{ userInfo?: { nickName, avatarUrl } }
```

**输出：**
```javascript
{
  success: true,
  openId: string,
  role: 'normal' | 'parent' | 'child',
  roleInfo: {
    role: string,
    description: string,
    icon: string,
    permissions: object
  },
  nickName: string,
  avatarUrl: string,
  bindCode: string | null,
  hasBindCode: boolean
}
```

#### 2. ensureParentBindCode
确保家长拥有永久 6 位绑定码。

**输入：** 无

**输出：**
```javascript
{
  success: true,
  bind_code: string,
  bind_code_create_time: date,
  role: 'parent',
  roleInfo: object,
  isNew: boolean
}
```

**角色校验：**
- 孩子角色不能生成绑定码

#### 3. bindToParent
孩子输入家长绑定码进行绑定。

**输入：**
```javascript
{ bindCode: string }  // 6 位数字
```

**输出：**
```javascript
{
  success: true,
  message: string,
  parentOpenIdMasked: string,
  childRole: 'child',
  parentRole: 'parent',
  isNew: boolean
}
```

**角色校验：**
- 家长角色不能绑定为孩子
- 一个孩子只能绑定一个家长

#### 4. getMyBindings
获取当前用户的绑定关系。

**输入：** 无

**输出：**
```javascript
{
  success: true,
  data: {
    asParent: [
      { openid, name, role, roleInfo, bind_time }
    ],
    asChild: [
      { openid, name, role, roleInfo, bind_time }
    ],
    self: {
      openid,
      role,
      roleInfo,
      name
    }
  }
}
```

#### 5. unbindChild
家长解绑孩子。

**输入：**
```javascript
{ childOpenId: string }
```

**输出：**
```javascript
{
  success: true,
  message: '解绑成功',
  childRole: 'normal' | 'child',
  parentRole: 'parent',
  childRoleUpdated: boolean,
  remainingChildrenCount: number
}
```

**角色处理：**
- 孩子解绑所有家长后：role 从 child 还原为 normal
- 家长解绑所有孩子后：role 保持 parent（保留绑定码）

#### 6. updateMyProfile
更新用户档案，支持角色切换。

**输入：**
```javascript
{
  nickName?: string,
  avatarUrl?: string,
  role?: string,
  setRole?: boolean,
  forceUpdateRole?: boolean
}
```

**输出：**
```javascript
{
  success: true,
  data: {
    user: object,
    role: string,
    roleInfo: object,
    updated: number
  }
}
```

**角色切换校验：**
1. 孩子角色且有家长绑定时，不能切换为其他角色
2. 家长角色且有孩子绑定时，不能切换为孩子
3. 切换到普通用户时，清空绑定码

## 共享工具模块

### _shared/userRoles.js

提供角色相关的常量和方法。

**导出内容：**
- `ROLE` - 角色枚举
- `ROLE_DESCRIPTION` - 角色描述
- `ROLE_ICON` - 角色图标
- `ROLE_PERMISSIONS` - 角色权限配置
- `getRoleInfo(role)` - 获取角色信息
- `isValidRole(role)` - 验证角色是否有效
- `canGenerateBindCode(role)` - 检查是否可以生成绑定码
- `canBindAsParent(role, parentCount)` - 检查是否可以绑定为家长
- `canBindAsChild(role)` - 检查是否可以绑定为孩子

### _shared/userUpsert.js

统一的用户档案 upsert 工具。

**导出内容：**
- `upsertUserProfile(db, openid, patch, options)` - 更新或插入用户档案
- `setUserRole(db, openid, newRole, now)` - 设置用户角色（含校验）
- `findUsersByOpenid(db, openid)` - 按 openid 查找用户
- `normalizeRole(role)` - 标准化角色值
- `getBindingStats(db, openid)` - 获取绑定统计

## 索引建议

### users 集合
```
1. openid_idx: { openid: 1 } unique
2. _openid_idx: { _openid: 1 } unique
3. role_idx: { role: 1 }
4. bind_code_idx: { bind_code: 1 } unique
5. role_openid_idx: { role: 1, openid: 1 }
6. create_time_idx: { create_time: -1 }
```

### binds 集合
```
1. parent_openid_idx: { parent_openid: 1, status: 1 }
2. child_openid_idx: { child_openid: 1, status: 1 }
3. parent_child_idx: { parent_openid: 1, child_openid: 1, status: 1 } unique
4. create_time_idx: { create_time: -1 }
```

## 角色流转图

```
                    ┌─────────────┐
                    │   新用户    │
                    │  (normal)   │
                    └──────┬──────┘
                           │
              ┌────────────┴────────────┐
              │                         │
              ▼                         ▼
    ┌─────────────────┐       ┌─────────────────┐
    │  生成绑定码     │       │  输入家长绑定码  │
    │  成为家长       │       │  成为孩子       │
    │   (parent)      │       │   (child)       │
    └────────┬────────┘       └────────┬────────┘
             │                         │
             │                         │
             ▼                         ▼
    ┌─────────────────┐       ┌─────────────────┐
    │  绑定孩子       │       │  家长解绑       │
    │                 │       │  → normal       │
    │  解绑所有孩子   │       │                 │
    │  → 保持 parent  │       │                 │
    │  (保留绑定码)   │       │                 │
    └─────────────────┘       └─────────────────┘
```

## 使用示例

### 前端调用示例

```javascript
// 1. 获取用户信息（含角色）
const userInfoRes = await wx.cloud.callFunction({ name: 'getUserInfo' });
console.log('当前角色:', userInfoRes.result.role);
console.log('角色信息:', userInfoRes.result.roleInfo);

// 2. 成为家长（生成绑定码）
const parentRes = await wx.cloud.callFunction({ name: 'ensureParentBindCode' });
if (parentRes.result.success) {
  console.log('绑定码:', parentRes.result.bind_code);
}

// 3. 绑定到家长（成为孩子）
const bindRes = await wx.cloud.callFunction({
  name: 'bindToParent',
  data: { bindCode: '123456' }
});

// 4. 获取绑定关系
const bindingsRes = await wx.cloud.callFunction({ name: 'getMyBindings' });
console.log('作为家长的孩子:', bindingsRes.result.data.asParent);
console.log('作为孩子的家长:', bindingsRes.result.data.asChild);

// 5. 更新角色
const updateRes = await wx.cloud.callFunction({
  name: 'updateMyProfile',
  data: { setRole: true, role: 'normal' }
});
```

## 数据清理

使用 `initDbCollections` 云函数清理脏数据：

```javascript
// 清理 role 为空的用户，统一设置为 'normal'
const cleanupRes = await wx.cloud.callFunction({
  name: 'initDbCollections',
  data: { action: 'init', cleanup: true }
});
console.log('清理的用户数:', cleanupRes.result.data.cleanup.cleaned);
```
